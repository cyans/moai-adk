## 📋 SPEC 개요

이 문서는 MoAI-ADK 프로젝트의 모든 SPEC(사양서)을 체계적으로 관리하는 인덱스입니다. 각 SPEC의 상태, 우선순위, 관계를 명확히 하여 프로젝트의 진행 상황을 한눈에 파악할 수 있도록 합니다.

## 🔗 SPEC 관계도

### 종속성 관계

```
SPEC-WINDOWS-COMPLETE-001 (완료)
    ↓
SPEC-STATUSLINE-CLEANUP-001 (완료)
    ↓
[다음 SPEC]
```

#### 상세 종속성 분석

**SPEC-WINDOWS-COMPLETE-001 → SPEC-STATUSLINE-CLEANUP-001**
- **TAG-WIN-001**: 플랫폼 감지 시스템 → 상태선 환경 감지 기반 제공
- **TAG-WIN-005**: 사용자 지정 설정 관리 → 상태선 설정 관리 기반 제공
- Windows 최적화 기술이 상태선 개선에 적용됨

| 연관 기술 | 관련 SPEC | 영향도 | 설명 |

| **Windows 인코딩 처리** | SPEC-WINDOWS-COMPLETE-001, SPEC-STATUSLINE-CLEANUP-001 | 높음 | UTF-8 강제 설정, 한글 경로 처리 공통 기술 |

## 📝 SPEC 관리 가이드

### 새 SPEC 생성 절차

1. **요구사항 분석**
   - 사용자 요구사항 또는 시스템 개선 필요성 파악
   - 기존 SPEC과의 종속성 관계 분석

2. **SPEC 초안 작성**
   - 기본 정보 작성 (ID, 이름, 상태, 우선순위)
   - 요약 및 목표 정의
   - 상세 사양서 작성

3. **상태 관리**
   - 초안 작성 시: `draft` 상태로 설정
   - 개발 시작 시: `in-progress` 상태로 변경
   - 완료 시: `completed` 상태로 변경

4. **인덱스 업데이트**
   - 이 인덱스 문서에 새 SPEC 추가
   - 종속성 관계 업데이트
   - 진행률 통계 갱신

### SPEC 수정 및 보관

#### 수정 규칙
- **중요 수정**: 버전 업데이트 (예: 1.0.0 → 1.1.0)
- **경미한 수정**: 버전 유지, 내용만 업데이트
- **상태 변경**: 명시적인 버전 관리 없이 진행률 반영

#### 보관 규칙
- **보관 대상**: `completed` 상태의 모든 SPEC
- **보관 위치**: `.moai/specs/` 하위 디렉토리 유지
- **정리 주기**: 분기별로 불필요한 SPEC 검토 및 정리
